# weather_tourism_pipeline

Учебный проект демонстрирующий:
- Сбор данные о текущей погоде для 5 городов через бесплатный API ключ на openweathermap.org
- Сохранение данные в RAW слой с правильной структурой папок в исходном JSON формате (COLLECT)
- Добавление в каждый файл метаданных
- Создание файла с логом сбора данных
 - Очистку и стандартизацию (CLEANED):
    1. Температуру - привести к целым числам
    2. Названия городов - стандартизировать на русском языке
    3. Время - привести к единому формату
    4. Валидация - проверить разумность данных (температура от -50 до +60°C)
- Создать файла с логом очистки с информацией:
    1. Количество исходных записей
    2. Количество очищенных записей
    3. Типы примененных правил
    4. Найденные проблемы
- Добавление к данным о погоде (ENRICHED):
    1. Географическую информацию:
        -  Федеральный округ
        - Часовой пояс
        - Примерное население города
    2. Туристическую привлекательность:
        - Сезон (весна, лето, осень, зима)
        - Комфортность погоды для туризма (создать свою шкалу)
        - Рекомендуемый тип активности
    3. Справочная информация:
        - Создать простой справочник городов в Excel/CSV
- Создание туристических витрин (AGGREGATED)
    1. Витрина 1: “Рейтинг городов для туризма” Файл: city_tourism_rating.csv
        - Рейтинг городов по комфортности погоды
        - Рекомендации по типу туров
        - Лучшее время для посещения
    2. Витрина 2: “Сводка по федеральным округам” Файл: federal_districts_summary.csv
        - Средняя температура по округам
        - Количество “комфортных” городов
        - Общие рекомендации
    3. Витрина 3: “Отчет для турагентств” Файл: travel_recommendations.csv
        - Топ-3 города для поездок сегодня
        - Города, где лучше остаться дома
        - Специальные рекомендации (взять зонт, теплую одежду и т.д.)

# Структура

data/ - Папка с данными (4 логических слоя)

    aggregated/ - Слой 4: Агрегированные витрины для бизнеса

      aggregation_log.txt - Лог создания витрин

      city_tourism_rating.csv - Витрина 1: Рейтинг городов по комфортности

      federal_districts_summary.csv - Витрина 2: Сводка по федеральным округам

      travel_recommendations.csv - Витрина 3: Отчет для турагентств (Топ-3 + рекомендации)

    cleaned/ - Слой 2: Очищенные и стандартизированные данные

      cleaning_log_20260518.txt - Лог очистки (количество записей, проблемы)

      weather_cleaned_20260518.csv - Очищенные данные: температура, влажность, ветер

    enriched/ - Слой 3: Обогащенные справочниками данные

      cities_reference.csv - Справочник городов (ФО, timezone, население, сезон)

      enrich_log.txt - Лог

      weather_enriched_20260518.csv - Данные + комфорт + рекомендации + сезон

raw/ - Слой 1: Сырые данные от API

      openweather_api/2026/05/16/ - Папки по дате сбора
        collection_log.txt - Лог сбора данных
        kazan_20260516.json - Сырые JSON от OpenWeather API по городу Казань
        moscow_20260516.json - Сырые JSON от OpenWeather API по городу Москва
        novosibirsk_20260516.json - Сырые JSON от OpenWeather API по городу Новосибирск
        saint_petersburg_20260516.json - Сырые JSON от OpenWeather API по городу Санкт-Петербург
        sochi_20260516.json - Сырые JSON от OpenWeather API по городу Сочи

scripts/ - Python-скрипты пайплайна
        collect_data.py - Сбор данных с OpenWeather API
        clean_data.py - Очистка и валидация данных
        enrich_data.py - Обогащение справочниками
        create_reports.py - Создание бизнес-витрин
        create_reference.py - Создание справочника городов

.env - Переменные окружения (API-ключи)
.gitignore - Игнорируемые файлы для Git
LICENSE - Лицензия проекта
README.md - Документация проекта
requirements.txt - Зависимости Python (pandas, requests и др.)
